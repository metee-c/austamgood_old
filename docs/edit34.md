🎯 เป้าหมายการตรวจสอบ (Objective)

ตรวจสอบความถูกต้องของระบบ WMS ในทุกมิติ ดังนี้

1️⃣ ความถูกต้องของสต็อก “ตั้งแต่เข้า → ออก → คงเหลือ”

รับเข้าแล้ว ต้องมีผลต่อยอดคงเหลือ

โอนย้าย (transfer) ต้อง

ปลายทาง +

ต้นทาง −

ปริมาณต้องเท่ากัน 100%

จัด / หยิบ / โหลด / ส่งออก ต้องหักสต็อกจริง

2️⃣ ตรวจสอบระดับ พาเลท (Pallet-level Integrity)

สำหรับ รหัสพาเลทเดียวกัน ต้องตรวจว่า:

ปริมาณสะสมจากประวัติเคลื่อนไหว
= ปริมาณในยอดสต็อกคงเหลือ / บ้านหยิบ / Dispatch

พาเลทเดียวกัน
❌ ห้ามมีอยู่หลาย Location พร้อมกัน (ยกเว้นมี Log ชัดเจน)

พาเลทที่ “ไม่มีการเคลื่อนไหว”
❌ ห้ามมียอดคงเหลือ

3️⃣ ตรวจสอบความถูกต้องเชิง Location / Flow

ตรวจจับกรณีต่อไปนี้:

สินค้าอยู่ใน Dispatch / Delivery-In-Progress
แต่ ยังไม่มี Log การออก

สินค้าอยู่ Receiving นานผิดปกติ

สต็อกคงเหลืออยู่ใน Location
แต่ ไม่มีประวัติเข้า Location นั้น

เคลื่อนไหววน (Loop)

A → B → A

ไม่มีเหตุผลทางธุรกิจ

4️⃣ ตรวจสอบ Movement Consistency

สำหรับ Move ID / Receive ID / ใบหยิบ / Loadlist:

ต้องมีความสัมพันธ์ครบ

1 Move ID

ต้องมีทั้ง “เข้า” และ “ออก”

ปริมาณต้องเท่ากัน

เอกสารอ้างอิง (IV / PL / FS / LD)

ต้องสัมพันธ์กับปริมาณที่ถูกตัดจริง

5️⃣ ตรวจสอบ Lot / วันผลิต / วันหมดอายุ

Lot เดียวกัน
❌ ห้ามมีวันผลิต / หมดอายุไม่ตรงกัน

Lot ที่หมดอายุแล้ว
❌ ห้ามอยู่ในบ้านหยิบ / Dispatch

สินค้าที่ไม่มี Lot
→ ต้องมีเหตุผล (เช่น Premium / ของแถม)

6️⃣ ตรวจจับ สต็อกติดลบ / ค่าผิดปกติ

ตรวจทุกชีทว่า:

คงเหลือแพ็ค / ชิ้น / น้ำหนัก
❌ ติดลบ

จำนวนเป็นทศนิยมในสินค้าที่ไม่ควรแตกแพ็ค

น้ำหนักไม่สัมพันธ์กับแพ็ค × ชิ้น

7️⃣ ตรวจสอบความสัมพันธ์ข้ามชีท (Cross-Sheet Reconciliation)

ตัวอย่าง:

ยอดใน ยอดสต็อกคงเหลือ
ต้องอธิบายได้จาก
→ ประวัติเคลื่อนไหว

สินค้าใน บ้านหยิบ / บ้านหยิบพรีเมี่ยม
ต้องมีต้นทางจาก Receiving หรือ Transfer

สินค้าใน จัดสินค้าเสร็จ / โหลดสินค้าเสร็จ
ต้องสะท้อนการ “ลดสต็อก” จริง
