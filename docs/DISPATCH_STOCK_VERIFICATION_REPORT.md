## 5. การทำงานของระบบ

### 5.1 Stock Movement Flow
```
Preparation Area → Dispatch → Delivery-In-Progress → Delivered
                      ↑            ↑
                      |            |
                   จัดเสร็จ    ยืนยันโหลด
```

### 5.2 Document Status Flow
```
draft → assigned → picking → completed → loaded
                                ↑          ↑
                                |          |
                          จัดเสร็จ    เข้า loadlist
                                      + ยืนยันโหลด
```

### 5.3 แท็บ "จัดสินค้าเสร็จ" แสดงเอกสารที่:
1. ✅ Status = `completed` (จัดเสร็จแล้ว)
2. ✅ ยังไม่ได้เข้า loadlist หรือ
3. ✅ อยู่ใน loadlist ที่ status = `pending` (ยังไม่ loaded)

### 5.4 แท็บ "จัดสินค้าเสร็จ" ไม่แสดงเอกสารที่:
1. ❌ Status = `assigned` หรือ `picking` (ยังไม่เสร็จ)
2. ❌ อยู่ใน loadlist ที่ status = `loaded` (โหลดเสร็จแล้ว)
3. ❌ อยู่ใน loadlist ที่ status = `voided` (ยกเลิกแล้ว)
