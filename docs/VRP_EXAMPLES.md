## การเปรียบเทียบอัลกอริทึม

### ทดสอบกับ 50 ออเดอร์ในกรุงเทพฯ

| อัลกอริทึม | จำนวนรถ | ระยะทาง (กม.) | เวลาคำนวณ (วินาที) | ต้นทุน (บาท) |
|-----------|---------|--------------|-------------------|--------------|
| Nearest Neighbor | 5 | 198.5 | 2.3 | 3,580 |
| Insertion Heuristic | 4 | 185.5 | 8.7 | 3,213 |
| Clarke-Wright Savings | 4 | 182.3 | 12.5 | 3,156 |
| Insertion + 2-opt | 4 | 178.9 | 15.2 | 3,098 |
| Savings + 2-opt | 4 | 176.2 | 18.8 | 3,045 |

**สรุป**: 
- **Nearest Neighbor**: เร็วที่สุดแต่ได้ผลลัพธ์แย่ที่สุด
- **Insertion**: สมดุลระหว่างความเร็วและคุณภาพ (แนะนำ)
- **Savings**: ได้ผลลัพธ์ดีที่สุดแต่ช้ากว่า
- **Local Search (2-opt)**: ปรับปรุงผลลัพธ์ได้ 3-5%

## Tips & Best Practices

### 1. การเลือกจำนวนโซน
```
จำนวนโซนที่เหมาะสม = ceil(จำนวนออเดอร์ / 10)

ตัวอย่าง:
- 20 ออเดอร์ → 2-3 โซน
- 50 ออเดอร์ → 5-6 โซน
- 100 ออเดอร์ → 10-12 โซน
```

### 2. การเลือกอัลกอริทึม
- **< 30 ออเดอร์**: ใช้ Insertion + 2-opt
- **30-70 ออเดอร์**: ใช้ Savings + 2-opt
- **> 70 ออเดอร์**: ใช้ Insertion (ไม่ใช้ Local Search)

### 3. การตั้งค่า avgSpeedKmh
- **ในเมือง (กรุงเทพฯ)**: 25-40 กม./ชม.
- **ชานเมือง**: 40-60 กม./ชม.
- **ระหว่างจังหวัด**: 60-80 กม./ชม.
- **ทางหลวง**: 80-100 กม./ชม.

### 4. การตั้งค่า serviceTime
- **ร้านค้าปลีก**: 10-15 นาที
- **ร้านค้าส่ง**: 15-20 นาที
- **โรงงาน**: 20-30 นาที
- **ศูนย์กระจายสินค้า**: 30-45 นาที

### 5. การใช้ Time Windows
- **Strict**: ใช้เมื่อลูกค้าเข้มงวดเรื่องเวลา (ร้านสะดวกซื้อ, ร้านอาหาร)
- **Flexible**: ใช้เมื่อยอมรับได้ถ้าส่งช้า 30-60 นาที
- **Ignore**: ใช้เมื่อไม่มีข้อจำกัดเรื่องเวลา

## การวิเคราะห์ผลลัพธ์

### ตัวชี้วัดความสำเร็จ

1. **Utilization Rate** (อัตราการใช้ความจุ)
   - ดีมาก: > 90%
   - ดี: 80-90%
   - ปานกลาง: 70-80%
   - ต้องปรับปรุง: < 70%

2. **Average Distance per Trip** (ระยะทางเฉลี่ยต่อเที่ยว)
   - ในเมือง: 40-60 กม.
   - ระหว่างจังหวัด: 150-200 กม.

3. **Average Stops per Trip** (จุดส่งเฉลี่ยต่อเที่ยว)
   - เป้าหมาย: 10-15 จุด
   - ถ้าน้อยกว่า 8 จุด: พิจารณารวมเส้นทาง
   - ถ้ามากกว่า 18 จุด: พิจารณาแบ่งเส้นทาง

4. **Cost per Delivery** (ต้นทุนต่อการส่ง)
   - ในเมือง: 50-80 บาท/จุด
   - ระหว่างจังหวัด: 150-250 บาท/จุด

## Troubleshooting Guide

### ปัญหา: ได้รถมากเกินไป

**สาเหตุ**:
- `vehicleCapacityKg` ต่ำเกินไป
- `maxStops` น้อยเกินไป
- `consolidationEnabled` = false

**แก้ไข**:
1. เพิ่ม `vehicleCapacityKg`
2. เพิ่ม `maxStops`
3. เปิด `consolidationEnabled`
4. เพิ่ม `detourFactor`

### ปัญหา: เส้นทางไม่เหมาะสม

**สาเหตุ**:
- อัลกอริทึมไม่เหมาะกับข้อมูล
- ไม่ได้ใช้ Local Search
- การแบ่งโซนไม่เหมาะสม

**แก้ไข**:
1. ลองเปลี่ยนอัลกอริทึม
2. เปิด `localSearchMethod` = "2opt"
3. ปรับ `zoneMethod` หรือ `numZones`
4. เปิด `useMapboxApi` = true

### ปัญหา: คำนวณช้าเกินไป

**สาเหตุ**:
- ออเดอร์มากเกินไป
- ใช้ 3-opt
- โซนมากเกินไป

**แก้ไข**:
1. เปลี่ยนเป็น `localSearchMethod` = "2opt" หรือ "none"
2. ลด `numZones`
3. เพิ่ม `maxComputationTime`
4. ใช้ `routingAlgorithm` = "insertion"

## สรุป

ระบบ VRP นี้สามารถปรับแต่งได้หลากหลายตามความต้องการ โดยมีจุดเด่นคือ:

✅ รองรับการจัดส่งทั้งในเมืองและระหว่างจังหวัด
✅ มีอัลกอริทึมหลายแบบให้เลือกใช้
✅ สามารถแบ่งโซนพื้นที่อัตโนมัติ
✅ คำนวณต้นทุนได้แม่นยำ
✅ รองรับ Time Windows
✅ ปรับปรุงเส้นทางด้วย Local Search

สำหรับการใช้งานจริง แนะนำให้:
1. เริ่มจากการตั้งค่าพื้นฐาน
2. ทดสอบกับข้อมูลจริง
3. ปรับแต่งพารามิเตอร์ตามผลลัพธ์
4. บันทึกการตั้งค่าที่ได้ผลดี
5. ใช้การตั้งค่าเดิมสำหรับสถานการณ์ที่คล้ายกัน
