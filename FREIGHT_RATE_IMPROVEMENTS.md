## 📋 สรุปการเปลี่ยนแปลง

### ✅ สิ่งที่ทำเสร็จแล้ว

#### 1. **API ใหม่: ดึงราคาตามจังหวัด**
**ไฟล์:** `app/api/freight-rates/by-province/route.ts`

**Endpoint:** `GET /api/freight-rates/by-province?province=ปทุมธานี&supplier_id=SVC001`

**ฟังก์ชัน:**
- ดึงราคาค่าขนส่งจาก `master_freight_rate` ตามจังหวัดปลายทาง
- กรองตามผู้ให้บริการ (supplier_id) ถ้ามีการระบุ
- คืนค่าราคาล่าสุดที่ตรงกับเงื่อนไข

**ตัวอย่างการใช้งาน:**
```typescript
const res = await fetch(`/api/freight-rates/by-province?province=ปทุมธานี&supplier_id=SVC001`);
const { data, error } = await res.json();
// data จะมี: base_price, helper_price, extra_drop_price, pricing_mode, etc.
```

#### 2. **ปรับปรุง Modal แก้ไขค่าขนส่ง**
**ไฟล์:** `components/receiving/EditShippingCostModal.tsx`

**การเปลี่ยนแปลง:**

##### A. เพิ่มคอลัมน์ "จุดอ้างอิงราคา" ในตาราง
- แสดงปุ่ม "เลือก" ในแต่ละแถวของออเดอร์
- เมื่อคลิกปุ่ม จะ:
  1. ตั้งค่าแถวนั้นเป็น "จุดอ้างอิงราคา" (highlight สีเหลือง)
  2. เรียก API ดึงราคาจากมาสเตอร์ตามจังหวัดของจุดนั้น
  3. นำราคาที่ได้มาใส่ในฟอร์มอัตโนมัติ

##### B. แสดงจังหวัดชัดเจน
- ดึงข้อมูลจังหวัดจาก `master_customer` ผ่าน `customerMap`
- แสดงในคอลัมน์ "จังหวัด" ด้วยฟอนต์ตัวหนา
- Fallback: ถ้าไม่มีข้อมูลจาก customer ให้ดึงจาก `stop_name`

##### C. เพิ่มคำแนะนำผู้ใช้

## 🎯 วิธีใช้งาน (User Flow)

### สำหรับผู้ใช้งาน:

1. **เปิด Modal แก้ไขค่าขนส่ง**
   - ไปที่ `/receiving/routes`
   - เลือกแผนเส้นทาง
   - คลิก "แก้ไขค่าขนส่ง"

2. **ดูตารางออเดอร์ในเที่ยว**
   - ตารางจะแสดงจังหวัดของแต่ละจุดส่ง
   - มีคอลัมน์ "จุดอ้างอิงราคา" ด้านขวา

3. **เลือกจุดอ้างอิงราคา**
   - คลิกปุ่ม "เลือก" ที่จุดที่ไกลที่สุด
   - ระบบจะดึงราคาจากมาสเตอร์อัตโนมัติ
   - ฟอร์มจะถูกกรอกด้วยราคาที่ดึงมา

4. **ปรับแต่งราคา (ถ้าต้องการ)**
   - แก้ไขราคาเริ่มต้น, ค่าเด็ก, ค่าจุดเพิ่ม
   - เพิ่มค่าใช้จ่ายอื่นๆ

5. **บันทึก**
   - คลิก "บันทึกทั้งหมด"

## ⚠️ สิ่งที่ยังต้องทำต่อ

### 1. แก้ไขข้อมูลในมาสเตอร์
**ปัญหา:** ข้อมูลใน `master_freight_rate` บันทึกชื่อร้านแทนจังหวัด

**ตัวอย่างข้อมูลปัจจุบัน:**
```sql
origin_province = "ดีเด่น เพ็ทมาร์ท สาขา นวมินทร์"  ❌ ผิด
destination_province = "ดีเด่น เพ็ทมาร์ท สาขา ตลาดมารวย"  ❌ ผิด
```

**ควรเป็น:**
```sql
origin_province = "กรุงเทพมหานคร"  ✅ ถูก
destination_province = "ปทุมธานี"  ✅ ถูก
```

**วิธีแก้:**
- แก้ไขหน้า Master Data (`/master-data/shipping-costs`)
- เปลี่ยน input เป็น dropdown เลือกจังหวัดจาก `THAI_PROVINCES`
- อัปเดตข้อมูลเก่าในฐานข้อมูล

### 2. ปรับปรุงฟอร์มเพิ่ม/แก้ไขราคา
**ไฟล์ที่ต้องแก้:**
- `components/forms/AddFreightRateForm.tsx`

**การเปลี่ยนแปลง:**
